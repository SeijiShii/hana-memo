# バグ調査レポート: 全 /api/* Vercel Function が hang

> **issue / slug**: 001 / vercel-handler-signature
> **重大度**: critical
> **再現可否**: 可 (決定的、ローカル vercel dev で 100% 再現)
> **報告日**: 2026-05-25
> **対象**: ../README.md

---

## 1. 症状

`vercel dev` (本番 Node runtime と同等) で `/api/health` 以外の全エンドポイントがレスポンスを返さず hang。クライアントは応答待ちのままタイムアウト (HTTP 000)。vercel dev ログに `The function "X.ts" is still running after 30s. (hint: do you have a long-running waitUntil() promise?)`。

## 2. 期待挙動

各エンドポイントが正常にレスポンスを返す (例: 認証ヘッダなしの `POST /api/storage/upload-url` → 401)。

## 3. 再現手順

1. `.env.local` に実キー投入 (or 不要)、`vercel dev --listen 3000` 起動
2. `curl -X POST http://localhost:PORT/api/storage/upload-url -d '{}'`
3. → 応答なし、タイムアウト (110s でも HTTP 000)
4. `curl http://localhost:PORT/api/health` → 200 (これだけ動く)

## 4. 再現環境

| 項目 | 値 |
|---|---|
| ランタイム | Vercel Functions (Node.js runtime)、`vercel dev` v54.4.1 |
| OS | WSL2 (Linux) — ただし OS 非依存 (Vercel 本番でも同様) |
| 該当コード | `api/**/*.ts` の 23 handler |

## 5. 影響範囲

- 影響機能: **API 層全体** (撮影→識別→保存 / 認証 / 課金 / 図鑑 / エクスポート 等、`health` 以外の全 endpoint)
- 影響範囲: 本番デプロイ時、ユーザーは一切の API 操作ができない (全 hang)
- データへの影響: なし (書き込みに到達しない)
- セキュリティ影響: なし (DoS ではなく自己機能不全)
- **本番リリースしていれば全機能停止の critical 障害**。デプロイ前のローカル runtime 検証で水際検出。

## 6. 暫定回避策

なし (handler 形の修正が必須)。

## 7. 関連ログ・決定的証拠

同一 `vercel dev` で最小ハンドラを比較:

| handler 形 | 結果 |
|---|---|
| `export default async function handler(req: Request): Promise<Response>` (素の関数, 引数1) | **HANG** (HTTP 000) |
| `export default { async fetch(req: Request) {...} }` (fetch Web Standard) | 200 OK (1.7s) |
| `export default { fetch: handler }` (fetch property) | 200 OK (1.9s) |
| `export default function handler(req, res)` (Node 形, health.ts) | 200 OK |
